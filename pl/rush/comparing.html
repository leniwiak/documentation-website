Porównywanie danych

Polecenie „cmp”

Do porównywania ze sobą informacji używa się polecenia „cmp”. To jemu w całości będzie poświęcony ten dokument.

Program „cmp” zawsze wymaga od użytkownika podania dokładnie trzech opcji: Pierwszego bloku danych do porówania, typu porówania i drugiego bloku danych do porównania.
Program „cmp” po zakończeniu swojej pracy wypuszcza do konsoli tekst „TRUE” i nadaje kod wyjściowy „0” jeśli porównanie jest prawdziwe lub tekst „FALSE” i kod wyjściowy „1” jeśli porównanie jest NIE prawdziwe.

Wartości numeryczne

Gdy porównywane są wartości numeryczne, dostępne są poniższe akcje porównawcze: 

A > B
           Większy niż...
A < B
           Mniejszy niż...
A => B / A >= B
           Większy lub równy od...
A =< B / A <= B
           Mniejszy lub równy od...
A = B / A == B
           Równy z...
A != B / A != B
           Różny od...

W przykładzie powyżej „A” i „B” oznaczają dwie opcje zawierające wyłącznie liczby.

Wartości tekstowe

Gdy porównywane są wartości tekstowe, dostępne są poniższe akcje porównawcze: 

A = B / A == B
           Równy z...
A != B / A != B
           Różny od...
A ~ B
           Zawierający...

W przykładzie powyżej „A” i „B” oznaczają dwie opcje zawierające dowolny tekst.

Porównywanie poleceń

Program „cmp” jest zdolny również do uruchomienia programów w ukryciu i porównania ich z innymi komendami lub danymi.
Każda komenda jednak emituje różne rodzaje informacji na wyjściu. Te informacje to:

  • Kod wyjścia: Liczba identyfikująca rodzaj błedu lub brak błędu (Numer zero oznacza brak błedu).
  • STDOUT: Tekst emitowany na standardowe wyjście, czyli kanał wyprowadzenia tekstu używany gdy wszystko idzie wedle planu.
  • STDERR: Tekst emitowany na wyjście ostrzegawcze, czyli kanał wyprowadzenia tekstu używany gdy występują błędy.

Aby określić, które z tych danych użytkownik chce porównać, należy użyć jednego z poniższych tekstów przed wpisaniem polecenia do wykonania. 

OUT:
    STDOUT
ERR:
    STDERR
CODE:
    Exit code

Przykłady użycia tej składni:

> cmp OUT:say "OK" = "OK"
TRUE
> cmp CODE:say "OK" = 0
TRUE
>

Użycie w praktyce

Polecenia „cmp” możesz użyć w środku logicznych bloków warunkujących takich jak „if”. 

if cmp 4 = 4 do
    say "They're equal!"
end

Powłoka Rush potrafi również wstawić zawartość zmiennych do polecenia „cmp”. 

set numer 4
if cmp $number = 4 do
    say "They're equal!"
end
